Detailed Description

A connected receiver.

Maintains an underlying UM receiver. When an LbmCtRcv is created, its full initialization is deferred until its start method is called.

com.latencybusters.lbmct.LbmCtRcv.LbmCtRcv ( )

Creates a Connected Topic Receiver object.

Creation of a CT Receiver object creates an underlying UM Receiver. Unlike a UM receiver, creation of a CT Receiver object is not a two step process. (A UM receiver requires first looking up a UM topic object, and then creating the receiver. A CT receiver combines the two steps in its start method.)

This constructor only creates the object. Its full initialization is deferred until its start method is called.

Initilaizes a CT Receiver object.

This is typically called immediately after the object is created (LbmCtRcv constructor). The start method can return before any connections are made.

When the application is finished using this CT Receiver, it should be stopped (stop API).

Parameters
inCtCT object to associate with this CT Source.
topicStrTopic string. This is the same as the symbol parameter when looking up a UM receiver topic object with LBMTopic.
rcvAttrAttributes object used to configure the underlying UM receiver. This is the same as the lbmrattr parameter when looking up a UM receiver topic object with LBMTopic.
rcvCbAn object implementing the LBMReceiverCallback interface. This is the same as the cb parameter when creating a UM receiver with LBMReceiver. It is used to deliver messages and other receiver-related events to the application. Note that the LBM_BOS and LBM_EOS events are filtered out and are not delivered to the application. The CT connection create and delete callbacks should be used instead.
connCreateCbCallback object invoked when a CT source connects to this CT receiver. Technically this parameter is optional (null should be passed if not needed), but its use is central to the Connected Topics paradigm.
connDeleteCbCallback object invoked when a CT source disconnects from this CT receiver. Technically this parameter is optional (null should be passed if not needed), but its use is central to the Connected Topics paradigm.
cbArgApplication-specific object to be associated with the connected receiver, which is passed to the three callbacks (rcvCb, connCreateCb, and connDeleteCb). Note that the CT Receiver maintains a reference to the cbArg object until the CT Receiver is stopped. This parameter is optional (null should be passed if not needed).
