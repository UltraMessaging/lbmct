Detailed Description

Connection object, mostly for internal use by CT.

Application must NOT create, start, or stop this object directly; it is fully managed by CT internally. There are only two public APIs associated with this object: getRcvConnCbArg and isHandshakeMessage. Those APIs should only be called from inside the receiver callback.

Typically an application ignores CT handshake messages, except that they consume sequence numbers. This API can only be called synchronously from inside the receiver callback.
