Detailed Description

A connected source.

Maintains an underlying UM source, which is available with the getUmSrc() method. When an LbmCtSrc is created, its full initialization is deferred until its start method is called.

com.latencybusters.lbmct.LbmCtSrc.LbmCtSrc ( )

Creates a Connected Topic Source object.

Creation of a CT Source object creates an underlying UM source. Unlike a UM source, creation a CT Source object is not a two step process. (A UM source requires first allocating a UM topic object, and then creating the source. A CT source combines the two steps in its start method.) Note that to send a message, the application must access the underlying UM source with the getUmSrc() method.

This constructor only creates the object. Its full initialization is deferred until its start method is called.

Initializes a CT Source object.

This is typically called immediately after the object is created (LbmCtSrc constructor). The start method can return before any connections are made. Any messages sent to the source prior to the first connection event may not be received by any receivers.

When the application is finished using this CT Source, it should be stopped (stop API).

Parameters
inCtCT object to associate with this CT Source.
topicStrTopic string. This is the same as the symbol parameter when allocating a UM source topic object with LBMTopic.
srcAttrAttributes object used to configure the underlying UM source. This is the same as the lbmsattr parameter when allocating a UM source topic object with LBMTopic.
srcCbAn object implementing the LBMSourceEventCallback interface. This is the same as the cb parameter when creating a UM source with LBMSource. It is used to deliver UM source events to the application. This parameter is optional (null should be passed if not needed). Note that the SRC_EVENT_CONNECT and SRC_EVENT_DISCONNECT events should be ignored. The CT connection create and delete callbacks should be used instead.
connCreateCbCallback object invoked when a CT Receiver connects to this CT source. Technically this parameter is optional (null should be passed if not needed), but its use is central to the Connected Topics paradigm.
connDeleteCbCallback object invoked when a CT receiver disconnects from this CT source. Technically this parameter is optional (null should be passed if not needed), but its use is central to the Connected Topics paradigm.
cbArgApplication-specific object to be associated with the connected source, which is passed to the three callbacks (srcCb, connCreateCb, and connDeleteCb). Note that the CT Source maintains a reference to the cbArg object until the CT Source is stopped. This parameter is optional (null should be passed if not needed).
